MAC下基于Hexo+Github.io搭建个人博客

Posté le 2018-03-14

大致步骤：

  1. 安装和配置Node.js环境
  2. 安装和配置Git环境
  3. Github账户的设置
  4. 安装和配置Hexo
  5. 将Hexo和Github page联系起来
  6. 发布文章
  7. 配置主题
  8. 添加sitemap和feed插件
  9. 添加404公益页面

1. 安装和配置Node.js

用于生成静态页面，node.js官网，傻瓜型下一步安装。
打开终端验证:

node -v
npm -v

2. 安装和配置Git环境

用于将本地Hexo内容提交到Github上。安装教程网上很多，不再这里赘述
打开终端验证：

git –version

3. Github账户的设置

默认已经拥有Github账户
新建代码库，在Repository name下填写：账户名.github.io
这里的账户名，建议跟Owner一致，不然后面会很麻烦。我在这里踩了很多坑。

创建成功后，点击Settings，下拉到GitHub Pages，你就可以看到博客地址可以访问了

4.安装和配置Hexo

新建一个文件夹，以hexo文件夹为例，进入文件夹

cd hexo

开始安装Hexo

npm install hexo -g

检查Hexo是否安装成功

hexo -v

初始化该文件夹

hexo init

当看到“Start blogging with Hexo!”，进展一切顺利。

接下来，npm将会自动安装你需要的组件

npm install

生成一套静态页面

hexo g

开启本地服务器

hexo s

浏览器输入http://localhost:4000/，即可看到网站首页

假如页面一直无法跳转，可能端口被占用。此时ctrl+c停止服务器，接着输入hexo server -p 端口号来改变端口号

hexo server -p 5000

访问5000的端口

看到如下页面，Hexo在本地的配置已经全部结束了。

5.将Hexo与Github page关联起来

5.1 配置Git个人信息

*如果是第一次，需要设置Git的user name和email

git config –global user. name “yourname”
git config –global user .email “yourmailname@gmail .com”

检查是否有.ssh的文件夹

cd ~/.ssh

生成密钥，连续三个回车，得到两个文件：id_rsa和id_rsa.pub

ssh-keygen -t rsa -C “yourmailname@gmail .com”

添加密钥到ssh-agent

eval “$(ssh-agent -s)”

添加生成的SSH key到ssh-agent

ssh-add ~/.ssh/id_rsa

登录Github，点击头像下的settings，添加ssh
新建一个New SSH key，将id_rsa.pub文件里的内容复制到key下的输入框

测试添加ssh是否成功，如果看到Hi后面是你的用户名，就说明成功了

ssh -T git@github .com

Hi 100gcode! You’ve successfully authenticated, but GitHub does not provide shell access.

如果ssh-key配置失败，
首先，清除所有的key-pair
ssh-add -D
rm -r ~/.ssh
删除你的github中的public-key

重新生成ssh密钥
ssh-keygen -t rsa -C “xxx@xxx .com”

接下来正常操作
在github上添加公钥public-key：
1、首先在你的终端运行 xclip -sel c ~/.ssh/id_rsa.pub将公钥内容复制到剪切板
2、在github在添加公钥时，直接复制即可
3、保存

测试：
在终端 ssh -T git@github .com

5.2 配置Deployment

在文件夹中，找到_config.yml文件，修改repo值

repo值是你github项目里的ssh

6. 发布文章

新建一篇博客，执行命令后，在 \ source \ _posts 下将会看到 article title.md文件

hexo new post “article title”

使用markdown编辑器打开就可以编辑文章了。文章编辑好后，运行生成、部署命令：

hexo g //生成
hexo d //部署

或

hexo d -g //在部署前先生成

如果出现以下的错误：

ERROR Deployer not found: git

则需要安装一个扩展

npm install hexo-deployer-git –save

7. 配置主题

Hexo官网主题页搜寻自己喜欢的主题。以hexo-theme-next为例

终端cd到blog目录下执行

git clone https: //github .com/iissnan/hexo-theme-next themes/next

将blog 目录下 _config.yml里的theme的值修改为next

终端cd到blog目录下执行

hexo clean //清除缓存文件（db.json）和已生成的静态文件（public）
hexo g //生成缓存和静态文件
hexo d //重新部署到服务器

找了几个主题，NexT的功能非常丰富，文档相当完善。
